# 전 세션 수용분 전수 시각 검증 — 코덱스 교차 검증용 (2차)

작성: 2026-07-21 (Claude Code). 1차 문서(`session_009_exhaustive_verification.md`, 009 한정)를 확장해
**4개 세션의 수용 pose 행 2,216개 전수**를 다룬다. 코덱스 1차 교차 검증의 지적
(완전 독립 블라인드 아님 → κ 미산출)을 반영해, 이번 문서는 **코덱스가 라벨 CSV를
열기 전에 몽타주만으로 독립 라벨링**할 수 있도록 자료를 분리 배치했다.

## 1. 방법 (1차와 동일 + 보강)

1. 프레임당 1셀 블라인드 몽타주(빨간 X=대체된 cleaned, 초록 O=수용, 랜드마크 약어만).
   006/007/008은 4×4 그리드(총 37장), 009는 3×3(11장, 1차 산출물).
2. **기계 사전 스크리닝 추가**: 수용 좌표가 같은 프레임의 유효 cleaned 랜드마크
   어디에서도 0.06 이상 떨어진 행을 자동 플래그(9행) → 전건 고배율 줌 재판독.
3. 몽타주 판독 중 의심 셀은 개별 줌 재판독 (예: 008 f1750 — 몽타주 약어 오독을
   줌으로 교정한 사례, 실제로는 핸드스탠드의 바닥 짚은 right_index 정답).

## 5. 독립 블라인드 κ 프로토콜 (3차 검증, 2026-07-21 개정)

**개정 사유**: 코덱스가 3차 실행을 정당하게 거부 — 2차에서 009의 원 라벨·경계
프레임과 008 표본을 이미 봤으므로, 같은 세션이 지금 라벨링하면 독립 블라인드가
아니며 κ가 과대평가됨. 초판의 "코덱스가 라벨 CSV를 보기 전에" 요건은 이 오염을
간과했음. **실행 상태: OPEN.**

### 유효한 3차의 실행 요건

1. **라벨러**: 원 라벨 CSV, 본 문서, 1차 문서(session_009_...), 관련 노션
   페이지를 **본 적 없는 새 세션 또는 새 라벨러** (사람 포함 가능).
2. **전달 자료**: `examples/output/kappa_round3_package/` **만** 전달 —
   `README_LABELING.md`(중립 안내), `montages/`(48장, 라벨 무표기),
   `labels_template.csv`(session/frame/landmark/page만 — 라벨·플래그·지표
   컬럼 없음), `zoom_tool.py`(수치 미표시 고배율 렌더).
3. **전달 금지(오염 경계)**: 라벨 CSV 일체, 검증 문서 2종, 노션 리포트,
   `crop_crosspass_agreement.csv` 직접 열람, 본 대화·코덱스 2차 세션.
4. **채점(별도 단계, 라벨링 종료 후)**: 원 라벨과 join → 3×3 혼동행렬,
   Cohen's κ, 신뢰구간. 오류 0건 시 error 클래스 추정 불가와 correct 편중에
   따른 κ 해석 한계(prevalence paradox)를 함께 명시.

## 6. 3차 실행 결과 (2026-07-21, 실행 상태: CLOSED)

새 Codex 세션이 클린 패키지만으로 2,216행을 전수 블라인드 라벨링
(`labels_round3_codex.csv`: correct 2,215 / borderline 1 / error 0, 빈 라벨 0).
채점(`round3_scoring_joined.csv`, 채점 주체: Claude — 라벨 생성이 아닌 기계적
대조이므로 오염 무관):

| 지표 | 값 |
|---|---|
| 원시 일치율 | **99.05%** (2,195/2,216) |
| 혼동행렬 | correct∩correct 2,195 / 원경계→correct 20 / 신경계(006 f846 right_thumb)→원correct 1 / **error 0·0** |
| Cohen's κ | **−0.0009** (부트스트랩 95% CI [−0.0024, 0.0000], 10,000회) |

**해석 (프로토콜 4항의 한계 명시 이행):**

1. **오류 클래스에서 라벨러 간 완전 일치** — 독립 블라인드 라벨러도 오류 0건.
   본 검증의 결정적 주장(오지정 없음 → 가드·margin 변경 불필요)이 라벨러
   독립적으로 확인됨. 단 양측 모두 0건이므로 error 클래스의 연관 통계는
   정의상 추정 불가(사전 명시된 한계).
2. **κ ≈ 0은 불일치가 아니라 prevalence paradox** — correct 편중(99%+)으로
   기대 일치(0.99053)가 관측 일치(0.99052)와 일치해 κ의 분자가 소멸.
   κ는 이 데이터 분포에서 정보량이 없으며, 사전 등록된 해석 한계 그대로.
3. **경계 클래스는 라벨러 주관적** — 원경계 20행과 신경계 1행의 겹침 0.
   "경계"(인접하나 중심 아님)는 라벨러 엄격도에 의존하는 주석이지 안정
   분류가 아님. 어느 라벨을 채택해도 의사결정은 불변(오류 아님 = 기각
   대상 아님).

**결론**: 수용 좌표에 오류가 없다는 핵심 결론은 이제 단일 라벨러 한계
없이 성립. 경계 주석의 주관성은 기록된 잔여 특성으로 남김.
