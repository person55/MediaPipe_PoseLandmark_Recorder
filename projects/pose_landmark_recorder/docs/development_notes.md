# Development Notes

## Current scope

Phase 1 focuses on recorded video input only:

- Read frames with OpenCV.
- Run MediaPipe Tasks Pose Landmarker in `VIDEO` mode.
- Store normalized `pose_landmarks`.
- Store estimated 3D `pose_world_landmarks`.
- Store transformed landmarks for Blender-oriented playback.
- Write `raw_pose.jsonl`, `raw_pose.csv`, and `metadata.json`.

Camera input, Blender import scripts, and PyInstaller packaging are planned after the first video-file path is verified.

## Pose cleaning workflow

Raw pose extraction and pose cleaning are separate stages.

`record_from_video.py` writes MediaPipe's direct output to `raw_pose.csv`, `raw_pose.jsonl`, `metadata.json`, and `preview.mp4`. These raw files are treated as source measurements and are not overwritten by the cleaning pipeline.

`clean_pose_data.py` reads the raw files and writes corrected outputs to `cleaned/`:

```plain text
cleaned_pose.csv
cleaned_pose.jsonl
frame_status.csv
quality_report.json
interpolation_report.json
corrected_preview.mp4
```

Cleaning policy:

- Long missing ranges are marked as `missing_long_gap` and are not interpolated.
- Short gaps up to `max_interpolate_gap` are filled with linear interpolation.
- Low visibility, low presence, frame-to-frame jumps, and bone length outliers are marked before interpolation.
- Smoothing uses pandas rolling mean and does not bridge long missing ranges.
- Corrected preview rendering uses 2D `pose` x/y coordinates, not `pose_world` 3D coordinates.
- `quality_flag` distinguishes measured, interpolated, unreliable, and missing rows for downstream Blender/C4D/After Effects importers.

## macOS GPU delegate check

MediaPipe's Python `BaseOptions` exposes both `Delegate.CPU` and `Delegate.GPU`.

On macOS, the GPU path initializes the TensorFlow Lite Metal delegate. Input frames must be passed as `SRGBA` for this recorder's GPU path. Passing `SRGB` caused MediaPipe's macOS `ImageFrame` to `CVPixelBuffer` conversion to fail with `unsupported ImageFrame format: 1`.

Verified smoke command:

```bash
python scripts/record_from_video.py \
  --input /tmp/dance_pose_recorder_smoke.mp4 \
  --output /tmp/dance_pose_recorder_gpu_check \
  --delegate gpu \
  --origin raw \
  --save-jsonl \
  --save-csv \
  --max-frames 3
```

Expected GPU initialization log:

```plain text
INFO: Created TensorFlow Lite delegate for Metal.
```

## Model file

The default model path is:

```plain text
models/pose_landmarker.task
```

The file is not generated by the codebase. Download a compatible MediaPipe Tasks Pose Landmarker `.task` model and place it there before running the recorder.
