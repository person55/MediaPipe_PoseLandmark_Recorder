# session_cpu_009 수용분 전수 시각 검증 (P1 프로토콜) — 코덱스 교차 검증용

작성: 2026-07-21 (Claude Code)
대상: `session_cpu_009` crop 재검출 수용 pose 행 **278개 전수** (91개 프레임)
프로토콜: 코덱스 교차 검증(2026-07-21)에서 권고된 블라인드 층화 라벨 방식

## 1. 방법

1. **전수 렌더**: 수용 278행을 프레임별로 묶어 91개 셀(프레임당 1셀)로 렌더.
   각 셀에 해당 프레임의 모든 수용 랜드마크를 표시 — 빨간 X = 대체된 cleaned
   좌표, 초록 O = 수용 좌표, 랜드마크 약어 라벨. 3×3 몽타주 11장.
2. **블라인드 원칙**: 이미지에 패스(forward/mirror/reverse)·합의 지표·변위
   수치를 표기하지 않음. 라벨러(Claude, 멀티모달 판독)는 순수 시각 근거로
   정답(correct) / 경계(borderline) / 오류(error) 3단계 판정.
3. **경계·의심 사례 확대 재판독**: 몽타주 해상도에서 판정 불가한 사례는
   개별 프레임 고배율 줌으로 재판정 (f8379 사례 참조).
4. **라벨-특징 결합**: 판정 후 패스·변위·교차 패스 거리와 결합해 판별력 분석.

### 라벨-특징 판별력 (코덱스 프로토콜 항목)

- 경계 행의 최근접 독립 패스 거리 중앙값 **0.0021** < 정답 행 **0.0097** —
  교차 패스 합의는 경계/정답을 구분하지 못함(오히려 역방향). 경계 사례는
  "모든 패스가 같은 주먹 가장자리에 동의"하는 유형이라 합의가 높음.
  → **합의 지표는 정답성 판별자가 아니라는 기존 결론(Loop 12 note)의 전수
  데이터 실증.**
- 오류가 0건이므로 PR/AUC 산출 불가(양성 클래스 부재) — 프로토콜상 판별력
  분석은 "판별할 오류가 없음"으로 종결.

## 3. 결론 및 최적화 판단

1. **009 수용 278행 전수에서 오지정·몸 밖·반대편 오류 0건.** 수용 좌표는
   전부 대상 신체 부위 위 또는 즉시 인접이며, 대체된 cleaned 좌표 대비
   개선(허벅지→실제 손, 엉덩이→실제 손 등) 또는 동등.
2. **가드·margin 변경 불필요 판정**: 오류 0건이므로 걸러낼 대상이 없고,
   경계 17행은 랜드마크 크기 이내의 오프셋으로 어떤 지표로도 정답과
   구분되지 않음(위 판별력 분석). 근거 없는 조정은 하지 않음 —
   Loop 11 보류 판정과 같은 원칙.
3. 코덱스 P1 caveat("수용 좌표의 위치 정확도는 표본 수준")은 009에 대해
   **전수 수준으로 해소**. 006_v2/007_v2/008의 수용분은 여전히 표본 수준
   검증 상태로 남음(기록).
